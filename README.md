## 🧠 Мой подход к разработке

| Принцип | Как я применяю |
|---------|----------------|
| **SOLID** | Использую DTO, репозитории, сервисы. Классы имеют одну зону ответственности. |
| **PSR-1, PSR-4, PSR-12** | Соблюдаю стандарты кодирования: `declare(strict_types=1)`, автолоадинг, отступы. |
| **DRY** | Выношу повторяющийся код в сервисы, трейты, хелперы. |
| **KISS** | Пишу простое решение под текущую задачу, не усложняя там, где не нужно. |
| **YAGNI** | Не добавляю интерфейсы и абстракции, если они не нужны прямо сейчас. |
| **DTO** | Использую типизированные объекты вместо массивов для передачи данных между слоями. |
| **Events / Listeners** | Выношу сайд-эффекты (уведомления, логи) из бизнес-логики. |
| **Repository + Service** | БД — через репозиторий, бизнес-правила — через сервис. |
